1. 命名统一原则
- 产品名称统一使用 Alertrix Dashboard。
- 站点名称统一使用 Pilot Monitoring Site。
- 页面内不再出现 403、CAT403、课程编号、作业编号等教学属性标题。
- 如果需要体现项目背景，可在答辩或文档正文中说明，但不建议直接放入界面主标题。
2. 当前版本总体评价
- 当前版本已经完成了基础信息架构：总览、系统摘要、快捷操作、设备卡、趋势预览、告警页、设置页都已具备雏形。
- 主要问题已经不再是“有没有页面”，而是“页面是否足够成熟、是否像正式 Web 监控平台”。
- 现阶段最需要提升的是信息密度、卡片完成度、图表表达力、操作入口的可点击感，以及各页面的内容丰满度。
3. 设计总原则
- Web 页面要更紧凑，不要让单张卡片占据过高的垂直空间。
- 每一块区域都要有足够的信息量，避免“大容器 + 两三行字”的占位感。
- 首页应优先回答四个问题：现在是否安全、是否有告警、哪个设备异常、下一步应该做什么。
- 所有页面都要保留操作闭环，让按钮、筛选、列表、详情入口看起来都像真实系统组件。
4. Dashboard 页面修改清单
4.1 顶部总览区
- 左侧统一改为：Alertrix Dashboard / Pilot Monitoring Site。
- 保留 Overall Status、Active Alerts、Devices Online、Last Sync 四个核心指标。
- 顶部色块高度适当降低，让首屏更多空间留给设备状态和趋势内容。
- 右侧增加真实按钮：View Alerts、Open Trends、Threshold Settings。
4.2 System Summary 区块
- 保留 Highest Severity、Active Alerts、Last Alert、Recommendation。
- 新增 Monitored Devices、Last Data Refresh、System Health。
- 建议文案：Highest Severity: Normal；System Health: Stable；Recommendation: All monitored parameters are within configured safe thresholds。
- 视觉上采用“浅色标签 + 深色值”的形式，压缩行距，减少空洞感。
4.3 Quick Actions 区块
- 不要继续使用横向铺满的浅色长条按钮，改为 2~4 个独立 action card。
- 建议按钮：View Alerts、Acknowledge Latest、Threshold Settings、View Device Status。
- 每个 action card 增加图标和一句简短说明，强化“可点击”和“可操作”感。
4.4 设备卡片区
- 保留 2x2 布局，但每张卡片高度减少约 35%~40%。
- 每张卡片统一展示：设备名、状态 badge、在线状态、当前值、单位、1h 变化、Warning 阈值、Critical 阈值、Last Update、Details / History 按钮。
- 重要数字不要贴在卡片底部，应放在中上区域，让用户一眼读到数值。
- 如条件允许，在卡片底部加入 mini trend 或 sparkline，提升完成度。
4.6 Risk Summary 与 Trend Preview
- Risk Summary 不建议只保留三条无数字的横线，改成 Critical / Warning / Normal 三个统计块，或至少在条形上显示数量。
- Trend Preview 不应只是一条平线，需要补上单位、时间轴刻度、阈值标签、Latest value、Peak value。
- 趋势区标题右侧保留 1H / 6H / 24H / 7D 切换，但样式应更像 segmented control。
4.7 Recent Alert Log
- 即使当前没有告警，也建议保留表头结构：Time、Device、Type、Severity、Status、Action。
- 标题右侧增加 View All。
- 空状态建议改为两行：No active alerts. / All monitored parameters are within configured safe thresholds。

5. Sensor Trends 页面修改清单
- 页面标题统一使用 Sensor Trends。
- 不要继续维持“一页四个大空白趋势框”的布局。
- 顶部增加筛选区：Device、Metric、Time Range、Refresh、Export；至少优先实现 Device 和 Time Range。
- 页面主体改成“一个主图 + 统计摘要 + 其他设备趋势缩略卡”的结构。
5.1 推荐页面结构
- 主图区域：显示当前选中的趋势图，包含折线、Warning 阈值线、Critical 阈值线、当前值标记。
- 图表摘要：Latest、Average、Peak、Min、Threshold Breaches。
- 下方缩略卡：Water Level、Vibration、Temperature、Humidity 四张小趋势卡。
- 页面底部可补一个简短历史记录表：Timestamp、Device、Value、Unit、Status。
6. Alert Center 页面修改清单
- 页面标题统一改为 Alert Center，不再使用 Alert Dashboard。
- 页面顶部增加 summary cards：Active Alerts、Critical Alerts、Acknowledged Today、Resolved Alerts。
- 筛选区保留 All / Normal / Warning / Critical，并可在后续追加 Active / Acknowledged / Resolved。
- 主体区使用表格或告警卡片列表，不要只停留在空状态提示。
